Technical Deep Dive

The technical implementation of ad refresh in header bidding involves several interconnected systems. At the protocol level, OpenRTB bid requests carry the signals that enable targeting, auction clearing, and performance measurement. The bid request fields — imp for impression details, site or app for publisher context, device for hardware and connectivity, user for identity and consent — each play a role in how ad refresh in header bidding functions in practice.

At InMobi Exchange, our product architecture addresses ad refresh in header bidding through multiple layers. The measurement layer tracks outcome metrics that validate our approach. And the API layer provides transparency to both publishers and demand partners about how ad refresh in header bidding affects their transactions.

The header bidding ecosystem — whether through Prebid.js for client-side, Prebid Server for server-side, or Amazon TAM — intersects with ad refresh in header bidding in important ways.
